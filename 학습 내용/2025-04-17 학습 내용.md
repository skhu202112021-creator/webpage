## DTO vs. Entity
DTO : 데이터, getter/setter를 가진다.
API 요청 응답(json)이나 DB 데이터를 프론트엔드나 백엔드와 교환할 때 사용한다.

Entity : 데이터베이스나 시스템에서 관리하고자 하는 것.

## sql injection(sql 주입) 공격
https://developer.mozilla.org/ko/docs/Glossary/SQL_Injection 참고.
우리가 웹사이트에 로그인을 하면 SQL에서는 어떤 일이 일어날까?

```
"SELECT Count(*) FROM Users WHERE Username=' " + txt.User.Text+" ' AND Password=' "+ txt.Password.Text+" ' ";
```

Users 목록에 입력한 Username와 Password가 있는지 확인하고 있다면 true를 반환하고 없으면 false를 반환해 로그인을 하는 명령어다.
Count(\*)는 행 번호를 의미한다.

만약 Username을 알기 쉬운 admin으로 하고 password를 passwd123이라고 하면 다음과 같다.

```
"SELECT Count(*) FROM Users WHERE Username=' admin ' AND Password=' passwd123 ' ";
```

이때 해커는 Username을 admin으로, password를 아래와 같이 입력한다.

```
Password=' anything 'or'1'='1 '
```

anything을 Magical String이라고 한다. 말 그대로 마법의 문자열.
admin 계정에 anything이라는 비밀번호가 없으니 false가 나온다.
그런데, or가 들어가서 둘 중에 하나만 참이면 로그인이 된다.
'1' = '1'이니 참이 되고 해커는 로그인을 할 수 있게 된다.
따라서 이 SQL문은 상당히 허술한 코드이다.

해결 방법 예시
```
$id = $_GET['id']

(1) $id = Stripslashes($id)

(2) $id = mysql_real_escape_String($id)
```
1. 작은 따옴표( ' )를 큰 따옴표( " )로 바꾼다.
2. 따옴표 앞에 /를 추가한다.
